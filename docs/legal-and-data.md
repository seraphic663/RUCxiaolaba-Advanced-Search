# 数据来源、隐私与合规说明

本文说明本项目如何获取数据、API 的性质、隐私风险和使用边界，不构成针对具体运营行为的法律意见。

## 数据是怎么获取的？API 是后门吗？

项目使用登录 RUC 小喇叭后产生的 `ys7_ysxy_session` cookie，请求微信小程序正常使用的文章列表、活跃列表和文章详情接口，再将帖子与评论写入本地 SQLite 数据库。项目不是抓取网页 HTML，也没有利用认证绕过或权限提升。

该接口应当理解为“小程序客户端内部业务 API”：它是小程序正常运行所使用的通信渠道，不是后门；但平台也没有将其声明为面向第三方开发者的开放 API。无 cookie 请求会返回“请先登录”，说明接口执行正常的会话认证。能够复现客户端请求不等于已经获得自动化批量调用许可。

## 项目获得授权了吗？是否合法？

当前仓库没有提供可验证的平台书面授权、官方 API 许可或真实数据公开许可，因此项目不能声称已经获得 RUC 小喇叭或中国人民大学授权。

也不能脱离具体使用方式笼统判断“整个项目合法”或“整个项目违法”：代码和合成 demo 可以用于开发测试；实际抓取是否合法取决于账号权限、平台规则、访问方式、请求强度和使用目的；保存、搜索或公开真实数据还涉及个人信息、用户内容著作权和数据安全责任。

持续抓取、全量扫描、公开部署或向他人提供真实数据前，应取得平台运营方对接口、频率、字段、用途、保存期限和展示范围的书面授权。平台授权也不当然替代个人信息处理依据和内容传播许可。

## 是否存在隐私泄露风险？

存在。帖子虽然可能显示为匿名，但正文、评论、稳定用户 ID、昵称、发帖时间和其他细节组合后仍可能识别个人。删除昵称或对 ID 做哈希通常只能降低风险，不能自动证明数据已经不可复原地匿名化。

公开真实数据库的风险尤其高：Git 仓库和公开制品可以被长期复制、检索和再次传播，还可能保留后来已经删除的内容，或包含健康、情感、财务、行踪、身份等敏感线索。因此，本仓库只提交虚构的 `demo/` 数据，不提交任何月份的真实帖子数据库。

MIT License 只覆盖项目贡献者有权授权的代码和文档，不授权真实帖子、评论、头像、用户数据、平台名称或其他第三方内容。

## 使用时必须注意什么？

- 只能使用本人合法取得且本人有权使用的 cookie，不得购买、借用、共享、窃取或诱导他人提供 cookie。
- 不得绕过登录、验证码、签名、限流、权限检查或其他访问控制。
- 无法确认平台授权范围时，只能使用仓库自带的合成 demo，不应连接真实接口。
- 只获取实现帖子搜索所需的文章列表、文章详情和评论，不得抓取私信、账户资料、手机号检查等非必要数据。
- 遇到认证失败、限流、异常响应或平台要求停止时必须停止，不得通过更换账号、代理或提高并发规避限制。
- 不得将数据用于身份还原、人肉搜索、骚扰、歧视、用户画像或对个人产生不利影响的自动化决策。
- 真实数据库不得提交到 Git、公共网盘或公开制品，应设置访问控制、最短保存期限、备份保护和删除机制。
- 公开展示、AI 处理、向第三方提供和公开数据库属于新的使用目的，需要分别评估，不能默认被一次授权覆盖。
- 用于研究、论文、课堂项目或公开服务时，还应确认所在机构的伦理审查、数据管理和信息安全要求。

## 一句话说明

> 这是一个通过小程序正常内部 API 获取帖子和评论的非官方搜索工具，不是后门，也不是获得官方许可的开放 API；仓库没有附带平台书面授权，只提供虚构 demo，任何真实抓取、保存或公开服务都应由使用者先确认授权、隐私和数据合规责任。
