# 安全、隐私与备份

## 敏感文件

以下文件不要提交到公开仓库：

```text
data/config.txt
data/admin_password.txt
data/posts.db
data/feedback.jsonl
data/checkin_count.json
```

其中：

- `config.txt` 包含小程序 cookie
- `admin_password.txt` 是后台密码
- `posts.db` 包含帖子、评论、展示 ID、真实 ID
- `feedback.jsonl` 可能包含用户主动提交的信息

## admin 权限

公开接口默认不返回评论用户 ID。

`/api/comments` 只有 admin 登录态才会返回：

```text
show_user_id
real_user_id
reply_show_user_id
```

普通用户只能看到公开评论内容和展示昵称。

## 匿名与真实 ID

字段含义：

```text
show_user_id  匿名展示 ID
real_user_id  真实用户 ID，0 通常表示匿名
```

admin 搜索支持：

- 匿名/实名筛选
- show_user_id / real_user_id 检索
- 评论用户 ID 检索

这些功能只应保留在 admin 登录态。

## 备份策略

### 小文件

适合高频备份：

```text
feedback.jsonl
checkin_count.json
admin_password.txt
config.txt
```

命令：

```bash
python scripts/backup_runtime.py --data-dir /app/data --keep 72
```

### posts.db

不建议频繁在同一个 5GB Volume 里备份完整 DB。

原因：

- 主库约 3.7GB
- Volume 只有 5GB 时，完整备份很容易占满空间
- SQLite WAL 运行中还可能额外占空间

更推荐：

- Railway Bucket
- S3
- R2
- 本地手动快照

### SQLite 在线备份

如果要备份 DB，应使用 SQLite backup API，而不是直接复制正在写入的 DB 文件。

项目脚本：

```bash
python scripts/backup_runtime.py --data-dir /app/data --include-db --keep 2
```

## 日常安全建议

1. admin 密码不要截图公开。
2. Railway Volume 不要放旧 CSV、临时上传切片、历史调试库。
3. cookie 过期后只更新 `config.txt`，不要写入代码。
4. admin 搜索结果截图前注意是否包含 ID。
5. 如果要公开演示，使用非 admin 页面。
