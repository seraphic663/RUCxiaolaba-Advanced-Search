# 爬虫与更新策略

## 唯一入口

当前只保留一个爬虫入口：

```text
crawler_db.py
```

它直接写 SQLite，不再写 CSV。

## 子命令

### new：补新帖

```powershell
python crawler_db.py new --db-path data\posts.db --pages 500 --min-pages 20 --stop-unchanged 300
```

默认接口：

```text
/article/article/lists
```

用途：

- 扫最新发布流
- 发现 DB 中没有的新 ID
- 发现评论数变化的帖子

### refresh：补新回复/活跃帖

```powershell
python crawler_db.py refresh --db-path data\posts.db --pages 500 --min-pages 20 --stop-unchanged 300
```

默认接口：

```text
/article/article/lists2
```

用途：

- 扫活跃流
- 更新近期有评论变化的旧帖

### backfill：补历史旧页

```powershell
python crawler_db.py backfill --endpoint lists --db-path data\posts.db --start-page 200 --pages 500 --min-pages 20 --stop-unchanged 600
```

用途：

- 从指定页开始向后扫
- 补历史缺口
- 低频运行

### detail-fill：指定 ID 修复

```powershell
python crawler_db.py detail-fill --db-path data\posts.db --ids 5014356,5018419
```

用途：

- 单帖修复
- 补异常帖子
- 手动验证 cookie/API

## 停止机制

当前核心参数：

```text
--pages            最多扫描页数
--min-pages        至少扫描多少页后才允许停止
--stop-unchanged   连续多少条不需要更新后停止
--max-details      最多抓多少个详情，0 表示不限
```

示例：

```text
--pages 500 --min-pages 20 --stop-unchanged 300
```

含义：

- 最多扫 500 页
- 至少扫 20 页
- 连续 300 条都无需更新后停止

这是偏保守的范围，适合线上定时任务。

## 写锁

`crawler_db.py` 会创建：

```text
data/posts.db.crawler.lock
```

作用：

- 避免多个 Railway Cron 同时写同一个 SQLite
- 避免 `database is locked`

即便有锁，也建议定时任务错峰运行。

## Railway Cron 建议

```text
new      每 15-30 分钟
refresh  每 30-60 分钟
backfill 每天或每周低频
```

示例：

```bash
python crawler_db.py new --db-path /app/data/posts.db --pages 500 --min-pages 20 --stop-unchanged 300
python crawler_db.py refresh --db-path /app/data/posts.db --pages 500 --min-pages 20 --stop-unchanged 300
python crawler_db.py backfill --endpoint lists --db-path /app/data/posts.db --start-page 200 --pages 500 --min-pages 20 --stop-unchanged 600
```

## 未来改进

### 只刷新计数字段

当前如果评论数不变，点赞、蹲蹲、浏览变化不会主动更新。

可以增加轻量模式：

```text
crawler_db.py metrics
```

逻辑：

1. 扫 `lists/lists2`
2. 如果 DB 已有帖子且评论数相同
3. 只更新 `star_count/trace_count/views/hot/updated_at`
4. 不抓详情，不重写评论

这样定期维护点赞和蹲蹲不会太慢。

### 更稳的历史补全

可以在 `crawl_state` 中记录 backfill 页码游标：

```text
backfill.lists.next_page
backfill.lists2.next_page
```

每次 Cron 从上次页码继续，避免重复扫同一段。
