# RUC小喇叭匿名机制分析报告

**报告日期**：2026-05-29
**分析方式**：mitmproxy 抓包 + API 逆向 + 实测验证

---

## 1. 结论（tl;dr）

**RUC小喇叭的匿名机制在匿名发帖模式下是有效的。** 匿名帖每次分配随机 `show_user_id`，无法跨帖关联。非匿名帖使用持久 ID，这是用户主动选择的行为，非平台漏洞。

---

## 2. 两种发帖模式对比

### 2.1 匿名模式

用户勾选"匿名"发帖时：

| 字段 | 行为 |
|------|------|
| `show_user_id` | **每次随机分配**（如 `<random_id_a>` → `<random_id_b>`） |
| `show_user_name` | 每次随机生成（如 `<random_name>`） |
| `real_user_id` | 始终返回 `0` |

**实测验证**：同一用户连续发两条匿名测试帖，两次的 `show_user_id` 完全不同（`<random_id_a>` ≠ `<random_id_b>`），无法通过 API 数据关联为同一人。

### 2.2 非匿名模式

用户不勾选"匿名"发帖时：

| 字段 | 行为 |
|------|------|
| `show_user_id` | **持久不变**（同一用户始终相同） |
| `show_user_name` | 用户的固定显示名 |
| `real_user_id` | 等于 `show_user_id` |

**说明**：非匿名发帖是用户主动选择的行为，持久 ID 属于平台正常功能，非隐私漏洞。

---

## 3. 发现的一些边缘情况（非漏洞）

### 3.1 "某同学1" = 多个不同人

平台默认显示名"某同学1"被多个不同用户共用。在抓包数据中发现至少 4 个不同的 `show_user_id` 都叫"某同学1"。这可能导致用户误判（以为所有"某同学1"评论是同一人），但**平台提供了 `show_user_id` 区分不同用户**，功能正常。

### 3.2 帖子中用户自行留下的联系方式

部分用户在帖子正文中留下 QQ、微信等联系方式（非平台泄露，是用户主动行为）。这与其他社交平台的情况一致。

---

## 4. 数据采集建议

当前爬虫（`spider.py`）未采集 `show_user_id` 字段。建议：

1. **匿名帖**：`show_user_id` 为随机值，采集意义不大
2. **非匿名帖**：`show_user_id` 为持久值，但用户主动选择非匿名，采集不涉及隐私问题
3. 无论匿名或非匿名，`real_user_id` 在匿名帖中始终为 0，是正确的隐私保护
